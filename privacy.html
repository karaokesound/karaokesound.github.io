1. General Information

This Privacy Policy describes the rules for processing personal data of users of the BakeBuddy mobile application.

The administrator of personal data is the owner of the BakeBuddy application.

service.bakebuddy@gmail.com

2. Scope of Data Collected

The application may process the following user data:

  • email address
  • username
  • user identifier (including Google ID when signing in via Google and Apple ID when signing in via Apple)
  • data shared via Google account (first name, last name, email address)
  • photos attached by the user to support requests
  • diagnostic data (error logs, crash data) stored on Azure server
  • technical data (IP address, system logs)

Providing data is voluntary, but necessary to use the application's features.

3. Methods of Data Collection

a) Registration in the app

During registration, the user provides an email address, username, and password. The password is stored exclusively in encrypted form (hash).

b) Google Sign-In

The application allows login via Google Sign-In. In this case, the application receives user data from the Google account (first name, last name, email address, Google ID) in accordance with the user's privacy settings. The legal basis for processing is the performance of a contract (Art. 6(1)(b) GDPR) and the user's consent.

c) Sign in with Apple

The application allows login via Sign in with Apple. In this case, the application receives the user's Apple ID and, optionally, their name and email address. The user may choose to hide their real email address, in which case Apple provides a private relay address. Data is processed in accordance with Apple's privacy settings. The legal basis for processing is the performance of a contract (Art. 6(1)(b) GDPR) and the user's consent.

d) Issue report form

The user may provide additional information when reporting problems via the in-app form. Optionally, the user may attach a photo to the report. Photos are stored only for the time necessary to handle the report and are then deleted.

e) Error logs and diagnostics

The application records error logs and crash data to ensure stability and improve application performance. This data is transmitted to and stored on Microsoft Azure servers. Logs may contain a user identifier but do not contain sensitive data.

4. Purpose and Legal Basis for Processing

  • enabling user registration and login – performance of a contract (Art. 6(1)(b) GDPR)
  • managing user account – performance of a contract (Art. 6(1)(b) GDPR)
  • handling support requests and communicating with the user – performance of a contract (Art. 6(1)(b) GDPR)
  • application security, system maintenance and improvement, error log processing – legitimate interests of the administrator (Art. 6(1)(f) GDPR)

User data is not used for automated decision-making or profiling.

5. Email Communication

The application sends email messages exclusively in the following cases:

  • sending a registration code
  • confirming a support request

Messages are sent via Google infrastructure (Gmail SMTP).

6. Data Retention

  • account data (email, username, identifier) – for the duration of the user's account
  • photos attached to support requests – only for the time needed to handle the request, then deleted
  • diagnostic data (error logs, crash data) – for up to 30 days
  • technical data (system logs, IP address) – for up to 30 days after account deletion

After account deletion, user data is deleted or anonymized in accordance with the above rules.

7. Data Security

  • password encryption (hashing)
  • server and communication security (HTTPS/TLS)
  • token-based authorization mechanisms (valid for a specified period)

8. Cookies and Similar Technologies

The mobile application does not use cookies. It may use session mechanisms and authentication tokens to maintain the user session and ensure proper login functionality.

9. Transfer of Data to Third Countries

User data may be transferred to countries outside the EEA, in particular to the United States, via:

  • Google LLC – in connection with Google Sign-In and email delivery via Gmail SMTP
  • Microsoft Corporation – in connection with Azure services (backend hosting, data storage, and error log storage)
  • Apple Inc. – in connection with Sign in with Apple

Data transfers are carried out on the basis of standard contractual clauses approved by the European Commission.

More information: Google Privacy Policy · Microsoft Privacy

10. Sharing of Data

Data is shared with third parties only to the extent necessary for the application to function:

  • Google LLC (login and email delivery)
  • Microsoft Corporation (Azure cloud infrastructure – hosting and diagnostics)
  • Apple Inc. (Sign in with Apple authentication)

Data is not sold or shared for marketing purposes.

11. User Rights

In accordance with the GDPR, the user has the right to:

Art. 15 GDPRaccess to their data
Art. 16 GDPRrectification of data
Art. 17 GDPRerasure of data
Art. 18 GDPRrestriction of processing
Art. 20 GDPRdata portability
Art. 21 GDPRobjection to processing

To exercise your rights, please contact the administrator:

service.bakebuddy@gmail.com

You also have the right to lodge a complaint with the President of the Personal Data Protection Office (UODO), ul. Stawki 2, 00-193 Warsaw, kancelaria@uodo.gov.pl.

12. Account Deletion

The user may delete their account at any time directly from within the application (in account settings). Deleting the account results in permanent deletion of personal data in accordance with the rules described in section 6.

If you experience any difficulties, please contact the administrator: service.bakebuddy@gmail.com

13. Children

The application is not intended for children under the age of 13, and the administrator does not knowingly collect personal data from such individuals. If the administrator learns that data of a child under 13 has been collected without verifiable parental or guardian consent, it will be deleted immediately.

14. Changes to the Privacy Policy

The privacy policy may be updated as needed. Users will be informed of any significant changes within the application or by email. The current version of the policy is always available at:

https://karaokesound.github.io/privacy
